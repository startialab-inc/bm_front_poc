# コンポーネントテスト実行

Vue コンポーネントの vitest テストを実行し、結果を報告するスキル。

## 引数

- **引数なし**: 全コンポーネントのテストを実行する
- **コンポーネント名あり**（例: `/component-test AppButton`）: 指定コンポーネントのテストのみ実行する

## 手順

### 1. 対象ファイルを特定する

**引数なし（全件実行）の場合**:
- `src/components/` 以下の全 `.spec.ts` ファイルが対象
- コマンド: `npm run test:run`

**コンポーネント名あり（絞り込み実行）の場合**:
- Glob で `src/components/**/{ComponentName}.spec.ts` を検索する
- ファイルが見つかった場合: `npx vitest run {ファイルパス}` を実行する
- ファイルが見つからない場合: ユーザーに「`{ComponentName}.spec.ts` が存在しません」と伝え、作成方法を案内する

### 2. テストを実行する

コマンド実行例（AppButton の場合）:

```bash
npx vitest run src/components/atoms/AppButton.spec.ts
```

全件実行の場合:

```bash
npm run test:run
```

### 3. 結果を報告する

以下の形式で結果をまとめて報告する:

**成功時**:
```
✅ {ComponentName}：{X} 件のテスト全て PASS
```

**失敗時**:
```
❌ {ComponentName}：{X} 件中 {Y} 件 FAIL

失敗したテスト:
- {describe 名} > {it 名}
  エラー: {エラーメッセージ}

考えられる原因:
- ...

修正案:
- ...
```

### 4. 失敗時の対応

失敗したテストがある場合は必ず以下を行う:

1. **エラーメッセージを読む**: 期待値と実際の値を確認する
2. **原因を特定する**: コンポーネントの実装変更・クラス名変更・props の変更が原因でないか確認する
3. **修正案を提示する**: テストコードの修正案またはコンポーネントの修正案をユーザーに提示する
4. **修正後に再実行**: ユーザーの承認を得て修正し、再度テストを実行して PASS を確認する
