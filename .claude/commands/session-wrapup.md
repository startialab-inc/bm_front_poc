# セッション締め作業

本セッションはこれで完了とします。本セッションで行った作業のうち、次セッションに引き継ぐ必要のある情報を整理し、CLAUDE.md をはじめ適切なドキュメントに記載してください。

## 手順

### 1. 本セッションの作業を振り返る

会話履歴・変更ファイルから、本セッションで完了した作業を把握してください。

### 1.7. Playwright MCP スクリーンショット・ログを整理する

1. `docs/playwright/` 内のファイル一覧を確認する（`.gitkeep` 以外）
2. 各ファイルについて以下を判断する:
   - **レビュードキュメント（`docs/review/` 内の Markdown）で参照されている** → `docs/review/screenshots/` に移動して Git 管理対象とする
   - **今セッション中に確認済みで今後不要** → 削除する（ユーザー確認不要）
3. プロジェクトルートおよび `.playwright-mcp/` に PNG / ログファイルが残っていないか確認する
   - **`.playwright-mcp/` 下のファイルはユーザー確認なしで即削除する**（ユーザーは実際の動作画面で確認できるため）
   - `docs/playwright/` へ移動が必要なファイルがある場合のみユーザーに確認する

### 2. CLAUDE.md を更新する

以下のセクションを必要に応じて更新してください：

- **現在のステータス**: 完了したセッション/タスクを追加
- **次のステップ**: 次回に引き継ぐタスクを整理（完了したものは ✅、残件は ⬜ を維持）
- **申し送り事項**: 技術的な注意点・引き継ぎ情報があれば追加
- **最終更新日時**: ファイル末尾の日付を更新する

### 3. 必要に応じて他のドキュメントも更新する

プロジェクトの構成に応じて以下を検討してください（TODO: プロジェクトの実態に合わせて更新）：

- `docs/review/{page}-review.md` — レビュー結果（セッション中に作成・更新した場合）
- `docs/figma-parts/_index.md` — UIパーツ解析進捗（UIパーツ解析セッションの場合）

### 4. 役目を終えたドキュメントを確認・削除する

以下の観点でドキュメントを見直し、不要なものは削除してください：

- **一時的な調査メモ・トラブルシューティング記録**: 問題が解決済みで今後参照しない文書
- **廃止された設計案・仕様書**: 実装方針が変更され、現行コードと乖離しているドキュメント
- **移行完了済みの手順書**: 作業が完了し、再実施の可能性がない手順書

削除する場合はユーザーに確認を取ってから削除してください。
不要かどうか判断できない場合はユーザーに確認してください。
