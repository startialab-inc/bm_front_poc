# セッション締め作業

本セッションはこれで完了とします。本セッションで行った作業のうち、次セッションに引き継ぐ必要のある情報を整理し、適切なドキュメントに記載してください。

## 手順

### 1. 本セッションの作業を振り返る

会話履歴・変更ファイルから、本セッションで完了した作業を把握してください。

### 2. 一時ファイルを整理する

`.claude/workflow-config.md` が存在する場合は `screenshot_temp_dir` の設定値を参照して一時ファイルを確認する。設定がない場合は `.playwright-mcp/` を確認する。

1. 一時ディレクトリ内のファイルを確認する
2. 各ファイルについて以下を判断する:
   - **レビュードキュメントで参照されている** → `docs/review/screenshots/` に移動して Git 管理対象とする
   - **今セッション中に確認済みで今後不要** → 削除する（ユーザー確認不要）
3. プロジェクトルートに PNG・ログファイルが残っていないかを確認し、不要なものは削除する

### 3. CLAUDE.md を更新する

以下のセクションを必要に応じて更新する:

- **次の作業**: 完了したタスクを ✅ に、残件を ⬜ のまま維持
- **申し送り事項**: 技術的な注意点・引き継ぎ情報を追記（完了した内容は削除してよい）

### 4. 進捗ファイルを更新する

`.claude/workflow-config.md` が存在する場合は `progress_file` の設定値（デフォルト: `docs/session-plan.md`）を更新する。

`docs/session-plan.md` が進捗ファイルである場合、以下の4箇所をセットで更新する:

1. **コンポーネント一覧テーブル** — 完了したコンポーネントの Phase C・Phase D 列を ✅ に更新
2. **Phase D タスクリスト** — 完了したコンポーネントの行を ✅ に更新
3. **「次フェーズ」の記述** — 次に着手するフェーズ・コンポーネントを明記
4. **最終更新日** — ファイル末尾の更新日を今日の日付に変更

### 5. memory を更新する

セッション中に生まれた以下の情報を memory に保存する:

- **フィードバック**: ユーザーに指摘・修正を受けたこと、「次回からこうして」というルール
- **プロジェクト進捗**: フェーズの完了・変更・次の着手先
- **判断・決定**: 設計上の判断や技術的な選択でセッションをまたいで引き継ぐべきもの

既存の memory ファイルと重複する場合は新規作成ではなく**既存ファイルを更新**する。

### 6. 役目を終えたドキュメントを確認する

以下の観点でドキュメントを見直し、不要なものはユーザーに確認の上削除する:

- **一時的な調査メモ**: 問題が解決済みで今後参照しない文書
- **廃止された設計案**: 実装方針が変更され、現行コードと乖離しているドキュメント
- **移行完了済みの手順書**: 作業が完了し、再実施の可能性がない手順書

### 7. 申し送り事項の確認

CLAUDE.md の「申し送り事項」は**次のセッションで Claude が最初に読む情報**です。
次のセッションで迷わず作業を開始できるよう、具体的かつ簡潔に記載されているか確認する。

## 参照設定ファイル

`.claude/workflow-config.md` に以下の設定を記述することで、このスキルの動作をプロジェクトに合わせられる:

```yaml
progress_file: docs/session-plan.md      # 進捗管理ファイルのパス
dev_command: pnpm dev                    # 開発サーバー起動コマンド
dev_url: http://localhost:8300           # 開発サーバーの URL
screenshot_temp_dir: .playwright-mcp/   # 一時スクリーンショットの保存先
```
