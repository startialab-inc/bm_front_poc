## 受け入れ手順

1. `run-story-tests` で検出された違反を確認する
2. 設計上意図的・修正不可能であることを判断する
3. 上記テーブルに追記する
4. `.storybook/preview.ts` の `a11y.test` 設定を `'todo'` のまま維持する
   （`'off'` にするとプロジェクト全体の a11y チェックが無効になるため）
