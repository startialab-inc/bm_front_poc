# 文書分類ルール

このプロジェクトでは、情報を4層に分類して管理する。
「どこに書くか迷ったとき」はこのドキュメントを参照すること。

## 4層構造

### 層1: `CLAUDE.md`（常時注入・最小限）

**目的**: セッション開始直後から最後まで確実に機能する最重要指示を置く。  
**目標行数**: 100 行以内（長いほど効果が低下するため）

**書くもの**:
- 技術スタック（バージョン付き）
- コマンド一覧（ビルド・開発・アセット生成）
- 行動原則（セッション全体を通して常に守るべきルール）
- ディレクトリ構造（簡略版）
- 次の作業（現在の進行状態を示すポインタ）

**書いてはいけないもの**:
- 完了済みのステータスログ（git 履歴で参照可能）
- 過去セッションの申し送り事項（セッション内の一時情報）
- 一般的なベストプラクティス（AI が既知の知識）
- コードスニペットの例示（1週間で陳腐化する）
- README と同じ内容のコピー
- 詳細な仕様・設計ドキュメント

---

### 層2: `.claude/rules/*.md`（ファイルパス連動注入）

**目的**: 特定ファイルを編集するときだけ注入されるコーディング規約を置く。  
セッション後半でも確実に機能する（対象ファイルへのアクセス時に再注入されるため）。

**フォーマット**:
```markdown
---
description: ルールの説明
paths:
  - src/components/**/*.vue
  - src/pages/**/*.vue
---

（ルール本文）
```

**`paths` の記法**: YAML 配列形式（VS Code 拡張機能との互換性のため配列形式を標準とする）

**書くもの**:
- コンポーネント固有の実装規約（命名規則・API 使用方法）
- テスト・検証規約
- セキュリティルール
- エラーハンドリング規約

**このプロジェクトの規約ファイル**:

| ファイル | 適用対象 | 内容 |
|---|---|---|
| `.claude/rules/vue-component.md` | `src/components/**/*.vue`, `src/pages/**/*.vue` | Vue SFC 実装規約 |
| `.claude/rules/verification.md` | `src/**/*.vue` | DOM/スタイル検証規約 |
| `.claude/rules/testing.md` | `src/components/**/*.spec.ts`, `src/pages/**/*.spec.ts` | テスト記述規約 |

---

### 層3: `docs/*.md`（参照情報・読みに行く情報）

**目的**: セッション開始時に読み込む詳細な参照情報を置く。  
CLAUDE.md には書かず、必要なときに `Read` ツールで参照する。

**書くもの**:
- アーキテクチャ・システム構成
- 設計仕様・デザイントークン
- 実装計画・進捗管理
- 外部サービス接続情報
- レビュー方針・評価観点

**このプロジェクトの参照ドキュメント**:

| ファイル | 内容 |
|---|---|
| `docs/architecture.md` | アーキテクチャ・MCP サーバー情報・外部サービス接続情報 |
| `docs/design-tokens.md` | デザイントークン一覧（色・レイアウト変数） |
| `docs/screens.md` | 対象画面一覧・画面遷移 |
| `docs/session-plan.md` | 実装計画（セッション区切り・実装状況） |
| `docs/review-plan.md` | レビューフェーズ方針・評価観点 |
| `docs/figma-parts/` | Figma UIパーツ解析結果 JSON |
| `docs/directory-structure.md` | ディレクトリ構造（src/ / docs/ / .claude/rules/） |
| `docs/storybook.md` | Storybook 作業方針・Story ファイル形式・動作確認手順 |
| `docs/a11y-known-violations.md` | Story テストで検出・受け入れ済みのアクセシビリティ違反一覧 |

---

### 層4: `.claude/projects/.../memory/`（セッション間引き継ぎ）

**目的**: セッションをまたいで引き継ぐべき判断・フィードバック・プロジェクト状態を置く。

**書くもの**:
- ユーザーから受けたフィードバック（「次回からこうして」というルール）
- プロジェクトの重要な方針変更
- 外部サービスの接続情報（トークン期限等）
- AI ツールへの操作上の注意事項

**書いてはいけないもの**（メモリから除外すること）:
- コードパターン・規約・アーキテクチャ（コードを読めばわかる）
- git 履歴・変更履歴（`git log` が正確）
- デバッグ手順・修正レシピ（コードとコミットコメントに残っている）
- セッション内の一時情報（作業中のタスク・現在の会話文脈）

---

## 更新タイミング

| 層 | いつ更新するか |
|---|---|
| `CLAUDE.md` | 技術スタック変更・新しい行動原則が生まれたとき |
| `.claude/rules/*.md` | コーディング規約が変わったとき |
| `docs/*.md` | 対応する設計・計画・仕様が変わったとき |
| `memory/` | セッション終了時（`/session-wrapup` スキル実行時） |
